## Why Isolade

<img src="https://isolade.com/icons/shield.svg" width="18" align="absmiddle">&nbsp; <strong>Isolation</strong>: Each agent gets its own microVM. Make use of best-in-class protection against all threats.<br>
<img src="https://isolade.com/icons/dashboard.svg" width="18" align="absmiddle">&nbsp; <strong>Many agents at once</strong>: You can switch between and steer them in one UI instead of juggling terminals.<br>
<img src="https://isolade.com/icons/lock.svg" width="18" align="absmiddle">&nbsp; <strong>Secretless</strong>: The VM only sees a placeholder. The proxy swaps in the real value for allowlisted hosts.<br>
<img src="https://isolade.com/icons/check.svg" width="18" align="absmiddle">&nbsp; <strong>Autonomous</strong>: No permission prompts. The agent can safely do anything with no permission prompts.<br>
<img src="https://isolade.com/icons/house.svg" width="18" align="absmiddle">&nbsp; <strong>Local</strong>: Runs on your own hardware. Your code, secrets and agent sessions can stay on your machine.<br>
<img src="https://isolade.com/icons/copy.svg" width="18" align="absmiddle">&nbsp; <strong>Shareable</strong>: Track your configuration in git. Share the same environment between all team members.<br>
<img src="https://isolade.com/icons/layers.svg" width="18" align="absmiddle">&nbsp; <strong>Multi-repo</strong>: Not bound to worktrees. Work with all repositories across the whole project at once.<br>
<img src="https://isolade.com/icons/zap.svg" width="18" align="absmiddle">&nbsp; <strong>Fast builds</strong>: Prewarm your images, enable shared caches, build in a fresh VM in just under a second.<br>
<img src="https://isolade.com/icons/bot.svg" width="18" align="absmiddle">&nbsp; <strong>Agent-agnostic</strong>: One interface for all agents. Claude Code and Codex today. Local models soon.

## Installation

One command for both macOS and Linux.

```bash
curl -fsSL https://isolade.com/install.sh | sh
```
